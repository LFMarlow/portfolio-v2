03

Approche

Ma philosophie de développement.

Code propre

Séparation des responsabilités, nommage explicite, architecture claire. Un fichier = une responsabilité. Le code doit être lisible par quelqu'un d'autre sans commentaires superflus.

Itératif

Avancer par petits incréments vérifiables. Tester, valider, commiter.

Versionné

Tout passe par Git. Commits, messages clairs, branches pour les features. Le code est toujours traçable et récupérable.

Documenté

README structuré, documentation des procédures, retours d'expérience sur les problèmes rencontrés.
